# Note App - デバッグ課題

このアプリケーションは、Railsのデバッグスキルを学習するための教材です。
**このアプリにはバグが含まれています**。エラーを見つけ、修正して、すべてのRSpecテストをパスさせることが目標です。

## 概要

- **リソース**: Note (ノート管理アプリ)
- **フィールド**: title (string), body (text)
- **機能**: CRUD操作（作成、読み取り、更新、削除）
- **テスト**: RSpec (Model spec, Request spec)

## セットアップ

### 1. 依存関係のインストール

```bash
bundle install
yarn install
```

### 2. データベースのセットアップ

```bash
rails db:migrate
```

### 3. サーバーの起動

```bash
rails s -p 8080
```

ブラウザで `http://localhost:8080` にアクセスしてください。

### 4. テストの実行

```bash
bundle exec rspec
```

## 取り組み方

このアプリケーションには複数のバグが含まれています。以下の手順でデバッグを進めてください。

1. **サーバーを起動してエラーを確認**
   ```bash
   rails s -p 8080
   ```

2. **エラーメッセージを読んで原因を特定**
   - サーバー起動時のエラー
   - 画面表示時のエラー
   - フォーム送信時のエラー

3. **バグを修正する**

4. **RSpecテストを実行して確認**
   ```bash
   bundle exec rspec
   ```

5. **すべてのテストがパスするまで繰り返す**

## エラーの種類

このアプリには以下のようなバグが含まれています：

### サーバー起動時のエラー
- ルート（root）の設定に問題がある
- ファイルが見つからないエラーが発生する

### 画面表示時のエラー
- リンクのパスが正しくない
- テンプレートで必要な変数が渡されていない
- データベースのテーブルが正しく作成されていない

### フォーム操作時のエラー
- リンクに必要なパラメータが不足している
- エラー処理が不適切
- テンプレートに構文エラーがある
- メソッドの呼び出し方が間違っている

## デバッグのヒント

- エラーメッセージをよく読む
- ファイル名やパスのスペルミスをチェック
- コントローラーのアクション内で必要な変数が定義されているか確認
- ビューファイルの構文エラーをチェック
- データベースのマイグレーションが実行されているか確認
- リンクヘルパーの使い方を確認（パラメータは正しいか？）

## 完了条件

すべてのバグを修正し、以下が達成できれば完了です：

- サーバーが正常に起動する
- すべてのページが表示できる
- ノートの作成・表示・編集・削除がすべて動作する
- RSpecテストが全てパスする（28 examples, 0 failures）

## 技術スタック

- Ruby 3.3.6
- Rails 8.0.3
- SQLite3
- RSpec 8.0.2
- Hotwire (Turbo + Stimulus)

エラーメッセージをよく読み、どのファイルのどの行でエラーが起きているかを確認しましょう。
Railsのエラーメッセージは詳細な情報を提供してくれます。
